### 5. AMBIENT & BEHAVIORAL UX RULES (CRITICAL FOR MEMORY CARE)

To ensure the system serves as a gentle, non-disruptive aid for cognitive decline, the Python backend must dynamically alter the generated HTML/CSS layout based on the time of day, and the Flutter app must handle safety guardrails.

#### A. Automated Day/Night Modes (Circadian Rhythm Support)
The Python HTML generator must automatically swap layouts based on the server's local time:
- **Day Mode (6:00 AM - 7:59 PM):** Light background, crisp black text. High contrast to ensure maximum readability during daylight hours.
- **Night Mode (8:00 PM - 5:59 AM):** Pure black background, stark white text (inverted mode). This dramatically reduces ambient light glare in a dark room, prevents sleep disruption, and serves as a heavy visual cue that it is bedtime.

#### B. Time-of-Day Contextual Greetings
The layout header must not just display a digital clock (which can cause confusion), but must explicitly state the time period in a giant visual badge:
- **06:00 - 11:59:** "Good Morning ☀️"
- **12:00 - 16:59:** "Good Afternoon 🌤️"
- **17:00 - 19:59:** "Good Evening 🌅"
- **20:00 - 05:59:** "Time for Sleep 🌙" (Paired with the dark mode layout).

#### C. Message Persistence and "Stale Data" Prevention
- **Expiration Rules:** Family messages can sometimes be forgotten by the senders. The Python backend must automatically expire any "Family Message" older than 24 hours. 
- **Fallback State:** If no message has been sent recently, the space must gracefully collapse or display a generic, reassuring
 default phrase (e.g., "We love you, grandma! Have a peaceful day.") rather than leaving an empty, broken-looking box.

#### D. Non-Tech Dashboard Safety Rails (Flutter App)
Because the family members using the app are non-tech-savvy, add these software constraints:
- **Character Limiters:** Restrict the Family Message text field to exactly 120 characters. This guarantees the text will always fit cleanly on the TRMNL screen without breaking the layout or overlapping other widgets.
- **Accidental Double-Tap Protection:** Disable the "Send to Screen" button for 5 seconds immediately after a tap, displaying a clear "Sending..." spinner so users don't spam the server.
