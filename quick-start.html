🩵 Ambient Care — Quick Start

Run the TRMNL Arabic memory day-clock on your machine. PowerShell, from the project folder.

🧠 1 · Run the backend

Terminal 1 — keep it running

The brain: aggregates data, renders the elder's screen, exposes the API.

cd backend
# first time only — creates your settings file
Copy-Item .env.example .env
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
Passcode & settings: open backend\.env and set AC_FAMILY_PASSCODE to your family login code (default is 1234). You can also change AC_DISPLAY_FONT and AC_TIMEZONE there. Restart the server after editing.

🔗 2 · Open these in your browser

While the backend is running on port 8000:

WhatURL
👵 Elder's screen (live) localhost:8000/api/trmnl.html
Exactly what the TRMNL device shows.
🛠️ API docs (try endpoints) localhost:8000/docs
Interactive Swagger UI for every endpoint.
❤️ Health check localhost:8000/api/health
Returns {"status":"ok"} if the server is up.
📱 Family dashboard localhost:8000/
The Flutter PWA — served at the root after a web build (see §4).

🧩 3 · Run the Flutter app (development)

Terminal 2 — alongside Terminal 1

Hot-reload dashboard, pointed at your local backend.

cd frontend
flutter run -d chrome --dart-define=API_BASE=http://localhost:8000

A Chrome window opens. Log in with your passcode, send a message, then refresh the Elder's screen tab to watch it appear. 🎉

📦 4 · One-server mode (like production)

Build the PWA once; the backend then serves it at / — everything on localhost:8000, no second terminal.

cd frontend
flutter build web --release
cd ..\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --port 8000

👀 5 · Quick screen preview (no server)

Render the day & night screens straight to HTML files.

cd backend
.\.venv\Scripts\python.exe smoke_test.py
Start-Process preview_day.html
Start-Process preview_night.html
💡 Stopping: press Ctrl + C in a terminal.  •  Reload flag: --reload auto-restarts on code edits (drop it for a normal run).  •  The database lives at backend\data\ambient_care.db.
